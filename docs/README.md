# Documentation Index

This folder contains the main documentation for running, understanding, and maintaining the Depolarization workflows.

## If You Are New To The Repository

Read in this order:

1. `getting_started.md`: install, configure, and run a first job.
2. `repository_layout.md`: understand where code, scripts, docs, and generated artefacts live.
3. `configuration.md`: review the runtime config structure and supported overrides.
4. `jobs.md`: understand job dependencies, inputs, and outputs.

## If You Want To Run A Specific Workflow

- `jobs.md`: one-page job reference for inputs, outputs, and config sections.
- `instrumental_effect.md`: detailed guide for the instrumental-effect pipeline.
- `configuration.md`: all configurable keys and task sections.

## If You Want To Modify The Code

- `development.md`: architecture, tests, and maintenance workflow.
- `functions.md`: generated symbol index across `src/code/`.

## Quick Links

- Setup and first execution: `getting_started.md`
- Repository structure: `repository_layout.md`
- Full task list and outputs: `jobs.md`
- Instrumental diagnostics: `instrumental_effect.md`
- REPL helper entrypoint: `DepolLib.depol_help()`

## Source Of Truth

- Runtime behavior is defined by the Julia code in `src/code/`.
- Default runtime examples come from `config/default.toml`.
- `functions.md` is generated by `scripts/generate_functions_doc.sh`.
